## 🎯 Overview

The `exporter` acts as Layer 4 of the OpenQ architecture stack. It solely handles async PDF, XLSX, and JSON rendering formats generated by completed multi-agent analysis jobs.

Because generating extremely large multi-page reports with Highcharts/Plotly SVG exports directly on an API response thread will rapidly exhaust worker capacity, this process relies heavily on decoupled offline computation.

---

## 🏗️ Architecture Design

Operatable as a strict offline output assembler.

- **Payload Rendering**: Upon job completion, an endpoint signals the exporter logic array. It queries Postgres for the specific dataset, graphs, executive summaries, and insight recommendations.
- **Tenant-Scoped Writes**: Using distinct directory isolation protocols, it writes generated structured reports to secure tenant-isolated directories in the underlying container volume mounting parameters.
- **Download Routing URLs**: Automatically generates scoped signed URLs natively transmitted back to the `api` routing module to surface in the frontend application UI matrices.
